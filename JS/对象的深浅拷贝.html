# 对象的深拷贝与浅拷贝

概念

深拷贝：占用同一块内存空间

浅拷贝：占用不同的内存空间

何时发生深浅拷贝

  1. 直接赋值计算(深拷贝)

    let obj = {a:1,b:2,c:{value:'apple'}};
let deepObj = obj;
deepObj.a = 10;
obj.b = 20;
obj.c.value = "mac"

  2. 对象序列化(浅拷贝)

let shallowObj = JSON.parse(JSON.stringify(obj))
obj.a = 10;
obj.c.value = "mac"
shallowObj.b = 20;

let shallowObj = {...obj}

深拷贝和浅拷贝优缺点互斥

  1. 优点：占用相同的内存空间，节省内存；作为一个变量的别名。
  2. 缺点：对于独立空间的变量发生相互影响，导致数据混乱。

浅拷贝的应用场景

  1. 在Vue中，借用pros传入的数据并对其进行写操作，应使用浅拷贝的方法赋值给第三方变量，避免直接修改了pros中的属性。
